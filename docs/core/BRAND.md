## Brand Identity

**Product Name:** ScrumHub
**Design Philosophy:** "Sober, Structured, Sophisticated"
**Reference UI:** Visual Studio Code
**Mascot:** Abstract geometric "S-Link" Dragon
**Catchphrase:** *Architecting Agility.*

---

## Brand Philosophy 
- To fully use the SCRUM methodology and documentation with the AI to power up and scale the project administrations and agile methodology.
- To give ideas order, structure and realism
- To be the developers and administrators best helper

## Coding philosophy
- Create a fully single-responsibility highly modular and scalable code 
- Use the AI Agents tools to develop highly intelligent and development ready code
- Create highly documented code for AI context and Feature real time status
- Create consistent code and use recognized frameworks

## Style philosophy
- Highly customizable and easy to understand UI/UX 
- Use simple and sober colors for professional looking UI
- Reduce use of excessive animations and only use to smooth interactions
- Keep views sober and prevent overstimulation and excess information per view

## User scope
### Description
The following talks about the considered uses and purpose ScrumHub can server for the different kind of users, both developers and non-developers.
The discussed features and use case is not by any means absolute or streamlined, it can be subjected to change and updates to adapt to real use case. 
### Users
**Product owners(PO)**: 
The application should allow the product owner to create backlogs easily and manage the project SCRUM lifecycle on multiple sprints with AI suggestions and analysis. 
*Features for PO*:
AI Assistant, Multiple backlog creation, report creation, statistics overview, collaborator and user management, product overview.
> The mentioned should also work for Scrum Master role

**Developers**:
Standalone and group developers should use ScrumHub to develop applications, manage project lifecycle, status and documentation state, streamline and track down burndown chart.
*Features for Developers*:
SCRUM Role management, GitHub connection and Repository AI Analysis, Git tools, Developers backlog, AI skills and personalized context, report system, Quest feature. 

**QA**:
Quality assurance should use the application for report creation and documentation supervision, accept and reject tasks completions and acceptance criteria for developers tasks.
*Features for QA*:
Multiple Backlogs, reports, AI reports, AI analysis, GitHub connection, acceptance criteria, chatroom.

**Non-developer users**
Non-developer user should be able to use this application to start on non-developer projects, they should use it for real life projects or as a life task manager with SCRUM methodology for goal keeping and report. 
*Features for Non-developer user*:
Quest feature, AI Assistant, backlog and task management, sprint reports and sprint retrospective, project creation and chatroom
